Configure Auditing Thresholds to Check for Abnormal Activity Patterns

Configuring auditing thresholds provides you a way to determine whether the amount of audit activity that has occurred over a specified period of time is over your normal operation levels. You can configure thresholds in the following key activity areas:

  • Alerts
  • Failed Logins
  • Security
  • DDL
  • Privileged User
  • Overall Activity

Once configured, you can quickly determine whether excessive or unexpected activity has occurred.

To configure activity thresholds:

  1. While on the SQL Server Activity Report Card, select any of the activity areas.
  2. Click the Threshold text above the graph on the activity report card.
  3. Enter the appropriate thresholds.
  4. Click OK.
