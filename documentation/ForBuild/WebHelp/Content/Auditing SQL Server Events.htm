Audit SQL Server Events

Auditing your SQL Server instances and databases is the first step in ensuring your SQL Server environment remains in continuous compliance with federal and corporate security and privacy policies. You can also generate reports on the audit data you collect, allowing you to demonstrate compliance on demand. For more information, see Reporting on audit data.

Auditing checklists

Use the following checklists to help you prepare your environment to successfully audit your SQL Server instances and databases. If you plan to audit virtual SQL Servers running in Microsoft failover clusters, see How to audit a virtual SQL Server instance for detailed installation and configuration tasks.
