How console security works

SQL Compliance Manager controls user access by leveraging the SQL Server logins that exist on the SQL Server instance hosting the Repository databases. When you start the Management Console, SQL CM automatically attempts to connect to the Repository. The Management Console validates your SQL Server privileges and restricts your access to the appropriate features. To be able to configure audit settings or report on audit data, your login must have the appropriate SQL Server privileges on the Repository databases.
