4.4 New Features

Audit the local SQL Server instance running the Collection Server on cluster

SQL Compliance Manager now supports auditing a virtual SQL Server instance including the local instance on a cluster running the Collection Server.

4.4 Fixed Issues

  • DE19724 An issue causing some users to receive multiple file parsing errors when upgrading from SQL CM 3.7 to SQL CM 4.3 does not occur when upgrading to SQL CM 4.4.
  • DE16024 SQL CM no longer incorrectly captures creating a linked server on SQL Server 2012 as "Create SQL Login" events. This activity is now captured as a "Create Linked Server" event.
  • DE16032 Users now can create a custom role on a SQL Server 2012 instance without the event returning as "invalid."
  • DE19827 This release corrects an issue that occurred after a user added a second node to a clustered repository, causing the SQL Compliance Agents to not process new files.
